# Project-Myriad

Project Charter: The Vision for Myriad
[cite_start]The project's vision is to be the definitive, open-source platform for manga and anime enthusiasts to manage, read, watch, and celebrate their collections. [cite_start]It is designed to seamlessly merge local media management with online discovery in a beautiful, intelligent, and community-enriched web experience.
The project is built on five core pillars:
 * [cite_start]Unified Library: The central nervous system that manages local files and online sources as one.
 * [cite_start]The Vault (Local Media Engine): The high-fidelity local library manager.
 * [cite_start]The Browser (Online Discovery Engine): The extension-based online aggregator.
 * [cite_start]The AI Core: The brain that powers recommendations, search, and "magic" features.
 * [cite_start]The Community Layer: The heart that connects users through shared interests.
Proposed Technology Stack
The following technologies are proposed for the project's development:
 * [cite_start]Frontend: React / SvelteKit / Vue
 * [cite_start]Backend: Node.js (Express/Fastify) / Go
 * [cite_start]Database: PostgreSQL / SQLite
 * [cite_start]Deployment: Docker, Vercel/Netlify
GitHub Setup and Management
[cite_start]GitHub will serve as the central headquarters for the entire operation.
Repository Creation
 * [cite_start]Create Repository: Go to GitHub and create a new public repository. [cite_start]It should be initialized with a README.md file.
 * [cite_start]Add License: Add a LICENSE file. [cite_start]The recommended license is the GNU GPLv3, as it ensures the project and its derivatives will always remain open source, which aligns with the spirit of community-driven projects.
 * [cite_start]Add .gitignore: Add a .gitignore file using a Node or Python template to prevent committing unnecessary files.
Project Management
 * [cite_start]Projects Board: Use the "Projects" tab to create a board. [cite_start]Create columns that match your roadmap, such as Backlog, To Do (Phase 1), In Progress, Needs Review, and Done.
 * [cite_start]Issues: Use the "Issues" tab as a brain for the project, turning every feature, bug, or chore into a distinct issue.
 * [cite_start]Labels: Use labels to organize issues by Priority, Type (bug, feature, etc.), and Phase.
 * [cite_start]Discussions: Enable the "Discussions" tab for brainstorming and Q&A to keep the "Issues" list focused on actionable tasks.
Automation with GitHub Actions
[cite_start]Use GitHub Actions as a quality control robot for the project. [cite_start]The first step is to create a simple workflow that runs a linter on every Pull Request. [cite_start]This is an easy win that keeps the codebase clean. [cite_start]Later, you can add actions to automatically run tests, build the application, and deploy it.

Project Myriad
The definitive platform for manga and anime enthusiasts.
[cite_start]Myriad is an open-source web application designed to be the ultimate home for your collections. [cite_start]It seamlessly merges local media management with online discovery in a beautiful, intelligent, and community-enriched experience.
[cite_start]
Core Features
 * [cite_start]Unified Library: Manage your local files and online sources all in one place.
 * [cite_start]The Vault: A powerful, offline-first manager for your local manga (.cbz, .cbr) and anime files.
 * [cite_start]The Browser: An extensible system to browse and read/watch from various online sources.
 * [cite_start]AI-Powered Discovery: Get recommendations based on art style, your reading habits, and more.
 * [cite_start]Community Focused: Share collections, build collaborative guides, and enjoy your hobbies with others.
